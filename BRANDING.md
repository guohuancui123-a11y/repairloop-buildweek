# Project Identity Guidelines

This document defines the naming and attribution conventions for Lobster AI System.

The goal is not to restrict forks. Forks are welcome. The goal is to keep the original project identity visible as the project spreads.

## Official Name

**Lobster**

This is the preferred short name for titles, headings, demos, screenshots, and social posts.

## Recommended Full Name

**Lobster AI System**

Use the full name when context is needed, especially in package listings, GitHub descriptions, articles, and fork attribution.

## Repository Slug

Recommended canonical repository slug:

```text
lobster-ai-system
```

Original repository:

```text
https://github.com/guohuancui123-a11y/lobster-ai-system
```

## Attribution Line

Recommended attribution text for forks and derivative works:

```text
Based on Lobster AI System, originally created by guohuancui123-a11y.
Original repository: https://github.com/guohuancui123-a11y/lobster-ai-system
```

## GitHub Repository Description

Recommended GitHub repo description:

```text
Lobster AI System — original local-first Python runtime repair tool. Run broken code, fix real errors, verify success. No API key, no cloud.
```

## Names to Avoid

Avoid names that remove the original Lobster identity or make a fork look like a separate unrelated original project.

Examples to avoid:

- `AutoFix AI`
- `Self-Healing Python`
- `Runtime Repair Bot`
- `BugFixer AI`
- `Python Repair System`
- `Code Healer`
- `Local AI Debugger`

If a fork needs a custom name, prefer a derivative form that keeps the origin visible, such as:

- `Lobster Fork`
- `Lobster Experimental`
- `Lobster Runtime Repair Lab`
- `<YourName>'s Lobster Build`

## Fork-Friendly Rule

You may fork, modify, publish, and experiment with this project under the MIT License.

Please keep a visible attribution link to the original Lobster AI System repository in your README, documentation, or release notes.
